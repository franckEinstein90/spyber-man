# Documentation Home

This folder centralizes repository-level and service-level documentation.

## Recommended Reading Order

1. **`repo-overview.md`** — understand all folders, responsibilities, and relationships.
2. **`backend-architecture.md`** — understand backend internals and processing flow.
3. **`backend-api-reference.md`** — endpoint-level contract and payload schema.
4. **`frontend-sample-one.md`** — understand how the frontend sample integrates with backend callbacks.
5. **`deployment-and-operations.md`** — run and operate locally and in server environments.
6. **`todos-and-technical-debt.md`** — known gaps and prioritized improvements.

## Document Scope

- The backend docs are intentionally detailed because backend code is the runtime source of crawling logic and callback delivery.
- The frontend sample docs describe both Streamlit UX and FastAPI callback receiver behavior.
- Technical debt items are grounded in current code observations and include practical remediation direction.
